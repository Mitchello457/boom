Module event

Subsystem for manging game events and subscriptions.

Event types are documented here.

| SIGNATURE | DESCRIPTION | | --------------- | ---------------------------------------------------------- | | ready () | Called by the map after all objects have been initialized. | | keydown (key) | Called by the display manager when key is pressed. | | keyup (key) | Called by the display manager when key is released. |

Functions

subscribe (event_name, callback, userdata)
Subscribe to an event. Makes a new subscription to 'event_name' events. 'callback' is called on event evaluation, with as the first arguments More arguments may be passed to the callback depending on the event. The returned object is the subscription. Call :destroy() on the subscription object to unsubscribe from the event.

Parameters:

  • event_name Event name to subscribe to.
  • callback Callback function for subscription.
  • userdata First arguments for callback (optional)

Returns:

    The subscription object.
push (event_name, ...)
Push a new event to the queue. Events are not evaluated immediately -- they are queued and then processed in the order they are received.

Parameters:

  • event_name Type of event to push.
  • ... Arguments to pass to subscriber callbacks.
next ()
Evalute the next event on the queue.

Returns:

    The event processed, or nil if no more events.
run ()
Process the event queue until no events remain.
